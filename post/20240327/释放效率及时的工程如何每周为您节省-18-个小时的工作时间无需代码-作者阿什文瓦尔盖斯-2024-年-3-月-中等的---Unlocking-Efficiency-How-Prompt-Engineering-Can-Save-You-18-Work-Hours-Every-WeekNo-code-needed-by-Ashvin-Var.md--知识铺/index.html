现在回到我们手头的任务。

虽然快速工程中使用了许多技术，从非常简单到非常复杂，但我们将重点关注相对简单但功能强大的技术，这些技术易于理解且易于应用，同时可以显着改进您的输出，并使您能够完成仅依靠 ChatGPT、Gemini 等时根本不可能完成的事情。

在开始之前，让我们先熟悉一些基本概念。

1. 代币 — LLMs 使用底层神经网络运行，因此要求输入为数字。这是通过将输入单词转换为称为标记的数字来实现的。这个过程称为标记化。一个 token 通常是一个单词的 3/4。

2.上下文窗口——上下文窗口是指模型可以用来执行单个操作的最大标记数，这包括作为查询/指令传递到模型中的标记以及由该模型。

不同的语言模型有不同的上下文窗口。

例如，如果您使用的模型的上下文窗口为 4097，并且您的查询或指令已经占用了 4000 个令牌，并且输出超出了剩余令牌限制，则模型很可能不会显示您的输出。

3. 温度——温度是指语言模型在生成输出时可以具有的可变性或创造力。温度值可以设置为最小值 0 到最大值 2 之间的任意值。对于大多数使用情况，温度值范围从 0.3 到 0.9 通常被认为是最佳的，而当您需要输出是确定性的并且需要输出时，则使用温度 0。可靠的。

随着温度设置得越来越高，LLM 可以生成多样化且富有创意的输出，但存在语无伦次甚至根本没有意义的风险。

4. Top p 和 Top k - 调整 top-k 通过限制对最可能的标记的选择来影响生成文本的多样性，而 top-p 通过控制所选标记的累积概率来影响生成文本的多样性和相关性采样期间。

二.上下文学习-零样本学习是一种模型将其知识从已见类别推广到未见类别的技术。例如，当模型成功地执行分类任务时，即使它最初没有接受过特定数据格式的训练。

大型模型通常更有能力进行零样本学习，并且这种能力通常会随着模型尺寸的减小而降低。然而，即使是大型模型有时也会表现不佳或在零样本学习中失败。

因此，通常认为一种更好的做法是在原始提示查询中包含一个或多个示例提示及其所需的输出，以促进更好的输出生成。这被称为一次性学习或少量学习，具体取决于提供的示例数量。

这是一次性提示的示例：

任务：写一首关于猫的简短有趣的诗。

这里曾经有一只名叫马蒂斯的猫，

他对纱线的热爱是无止境的。

带着顽皮的蔓延，

留下蓬松的白色碎片的混乱。

按照相同的押韵方案 (ABAB)，写一首关于狗的简短有趣的诗。

该提示为模型提供了所需格式（简短、有趣的诗歌）和押韵方案的示例。然后，模型可以使用这些信息来生成自己对狗的创造性反应。

尽管情境学习有多种应用，但最常见的两个用途是

1）给出模型必须执行的任务的示例。

  1. 提供模型上下文作为其输出的基础。此上下文可以是任何内容，从用于指导 LLM 响应的单个句子，如果上下文窗口允许，甚至可以是整篇文章或整本书。

截至撰写本文时，Google 的 Gemini 提供了 128,000 个令牌的上下文窗口，而 OpenAI 的免费模型根据特定模型提供多达 16,385 个令牌上下文窗口。

三．思维链推理 - LLMs 在需要根据常规提示进行逻辑推理的复杂任务上通常表现不佳。在这种情况下，您将能够让模型分解任务并使用称为“思维链推理”的提示技术进行逻辑思考。通常，这可以通过添加类似指令来简单地实现

“让我们一步步思考”

“分步骤描述你的推理。”

“在回答时解释一下你的思维过程。”

“将你的答案分解为中间步骤。”

“告诉我你是如何得出答案的。”

或者您可以尝试以不同的方式表达提示，而不是直接提出问题，您可以将其框架为需要推理的任务。

例如，“一个学生有 12 个苹果，送给朋友 5 个。学生还剩下多少个苹果？解释一下寻找答案的步骤。”

 一般提示

  • 不要假设模型知道您的要求，始终包含您在引出响应时与人类一起使用的短语，以指导他的思维或思维过程（例如，

    “让我们一步一步思考”），不是因为模型可以像人类一样思考，而是因为您使用的短语会出现在训练数据中的重复模式中，其中上下文与您希望模型运行的上下文相匹配。

    这会以这样的方式驱动结果，让你认为该模型具有类似人类的理解和常识，但事实并非如此。

  • 清晰的提示引导人工智能模型达到预期的结果，就像精心编写的地图引导旅行者到达目的地一样。

  • 当您开始设计提示时，您应该记住，这是一个迭代过程，需要进行大量实验才能获得最佳结果。您可以从简单的提示开始，然后不断添加更多元素和上下文，以获得更好的结果。

    因此，一路重复你的提示至关重要。当您开始尝试时，您会看到许多示例，其中特异性、简单性和简洁性通常会给您带来更好的结果。

  • 幻觉——一定是那些油炸服务器发出的烟雾😄——众所周知，LLMs会产生幻觉——我所说的幻觉是指答案中的化妆品。如果您有兴趣了解更多关于为什么会发生这种情况的信息，请访问 https://acesse.dev/u3i2z 。

    更重要的是，您需要记住的是，LLMs有时会伪造信息，尤其是在上下文窗口有限或提示不明确的情况下，因此如果需要，最好坚持使用传统搜索引擎是简单的信息检索，但如果您决定向 LLM 询问事实信息，请确保在使用或共享输出之前对输出进行事实检查。

 要点

  • 通过精心设计清晰且信息丰富的提示，您可以释放大型语言模型 (LLMs) 的全部潜力，并让它们执行您可能认为不可能的任务。
  • 基本提示元素：精心设计的提示结合了指令、上下文、输入数据和输出指示器，以引导 LLM 实现所需的结果。
  • 即使是基本的提示工程技术，例如分配角色、提供上下文和使用上下文学习，也可以显着提高 LLM 输出的质量和相关性。
  • 不要只是告诉 LLM 要做什么，而是提供线索和上下文来帮助它推理任务。思想链推理等技术可以鼓励逻辑反应。

 结论

对于任何与 LLMs 互动的人来说，快速工程都是一项变革性技能。通过了解如何制作有效的提示，您可以利用人工智能的力量来提高您的工作效率，开启新的创意途径，并从信息中获得更深入的见解。

当您踏上快速工程之旅时，请记住：清晰、背景和一些实验是释放人工智能助手真正潜力的关键。
