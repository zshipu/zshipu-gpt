 提示设计的力量

 介绍：

我花了很多时间研究和使用大型语言模型（LLMs），它们就像理解人类语言的超级智能程序。我做了很多研究，并在实际项目中尝试了不同的想法。

在这篇博客中，我想谈谈我学到的一些重要的东西：如何给这些LLMs正确的指示，以便他们可以做有用的事情。这就是所谓的提示设计，它很重要。

我将分享我从自己的经历中学到的东西，并向您展示它如何在我们与LLMs交谈时对我们的理解和回应方面产生重大影响。让我们一起潜入并探索吧！

什么是大型语言模型？

大型语言模型（LLM）是一种先进的软件系统，旨在像人类一样理解和生成语言。通过对来自互联网等各种来源的大型数据集进行广泛的训练，LLMs学习语言的结构和模式。

这使他们能够生成与人类语音非常相似的连贯文本。 LLMs 广泛应用于自然语言处理、翻译和内容创建等领域，改变了我们与语言相关技术交互的方式。

什么是提示工程？

提示工程是为大型语言模型 (LLMs) 创建指令以指导它们生成文本或响应的过程。

它涉及设计清晰有效的提示，提示 LLM 产生与特定任务或情况相关的所需输出。

提示工程的目标是通过精心设计提示来提高 LLMs 的性能，从而产生准确且适合上下文的响应。此过程会考虑预期任务、受众和主题等因素。

总体而言，即时工程对于优化 LLMs 的功能并在各种应用程序（包括自然语言理解、翻译和内容创建）中实现预期结果至关重要。

 提示类型：

语言模型提示是结构化输入，用于指导从 LLM 模型生成所需的输出。这些提示的复杂性和特异性各不相同，可满足不同的任务要求和输入数据可用性。以下是主要类型：

 零射击：

在零射击提示中，大型语言模型 (LLM) 的任务是在不提供任何具体示例或训练数据的情况下生成响应。该模型预计将利用其预先存在的知识来生成适当的响应。

 例子：

提示：“根据患者提供的症状诊断病情。”

输出：患者的症状表明可能诊断为肺炎。可能需要进一步的检查，例如胸部 X 光检查和血液检查来确认诊断。

说明：在此场景中，LLM 会在没有任何特定示例或训练数据的情况下生成诊断。它仅依靠现有的医疗状况和症状知识来做出诊断。

 一枪：

一次性提示涉及向 LLM 提供所需任务的单个示例或实例。该模型预计将根据该单一输入生成响应。

 例子：

提示：根据患者的病史和当前症状，诊断病情。

输入：患者有哮喘病史，并表现为气短、喘息和咳嗽。

输出：根据患者的哮喘病史以及当前的呼吸短促、喘息和咳嗽症状，可能的诊断是哮喘恶化。

说明：仅提供一个示例，LLM 使用输入数据来诊断患者的病情。它利用给定的信息做出明智的诊断。

 几枪：

很少有镜头提示为 LLM 提供所需任务的一些示例或实例。该模型从这些有限的示例中学习以生成响应，从而使其能够根据提供的实例进行调整和概括。

 例子：

提示：“根据患者情况，根据提供的症状和病史预测诊断。”

 输入：

1. 患者出现发烧、喉咙痛、淋巴结肿大等症状。病史包括最近去过热带地区。”

2.患者主诉腹痛、恶心、呕吐。病史表明以前诊断过阑尾炎。”

输出：根据第一位患者的症状和最近去过热带地区，可能的诊断是急性病毒性咽炎。对于第二名患者，症状和病史表明阑尾炎可能复发。

说明：通过提供的一些示例，LLM分析每个患者的症状和病史以做出诊断。它从这些实例中学习，以概括和预测类似案例的可能诊断。

总之，每种类型的提示（零样本、一次样本和少量样本）都可以应用于不同级别的输入数据，范围从没有示例到单个示例或几个示例。

LLM 相应地调整其诊断，利用其预先存在的知识并从提供的示例中学习来生成准确的评估。

 设计提示框架：

提示框架是一种结构化方法，用于设计提示以指导语言模型生成所需的输出。

这些框架提供了一种系统的方法来定义模型产生相关响应所需的输入格式、任务要求和上下文信息。

以下是一些常见的提示框架：

RTF（角色、任务、格式）提示框架是一种用于设计大型语言模型 (LLMs) 提示的结构化方法。

它由三个部分组成：

角色：角色组件定义 LLM 在生成响应时应采用的视角或角色。这有助于将提示置于上下文中并指定应生成响应的所需观点。

任务：任务组件概述了LLM预期要完成的具体目标或任务。它描述了提示的目标或目的，指导 LLM 生成相关且信息丰富的响应。

格式：格式组件指定响应应遵循的结构或格式。它定义了有关生成文本的组织、样式或布局的任何特定要求，确保输出的一致性和清晰度。

CTF（上下文、任务、格式）提示框架是另一种用于设计大型语言模型提示的结构化方法 (LLMs)

它由三个关键组件组成：

上下文：上下文组件提供与当前任务相关的背景信息或上下文。它通过提供 LLM 在生成响应时应考虑的任何必要的上下文或详细信息来为提示做好准备。

RASCEF（角色、操作、步骤、上下文、示例、格式）提示框架是一种用于设计大型语言模型 (LLMs) 提示的结构化方法。

它由六个关键部分组成：

角色：角色组件定义 LLM 在生成响应时应采用的视角或角色。它指定了响应所针对的实体或个人，或者应该生成响应的观点。

操作：操作组件概述了 LLM 预期执行的特定操作或任务。它描述了 LLM 应该如何响应提示，指导其行为或功能。

步骤：步骤组件将任务或操作分解为更小的、连续的步骤或阶段。它提供了一种完成任务的结构化方法，有助于组织响应生成过程。

上下文：上下文组件提供与当前任务相关的背景信息或上下文。它通过提供 LLM 在生成响应时应考虑的任何必要的上下文或详细信息来为提示做好准备。

示例：示例组件通过提供特定示例或场景来说明操作中的提示框架。它演示了角色、操作、步骤和上下文组件如何协同工作以指导响应的生成。

格式：格式组件指定响应应遵循的结构或格式。它定义了有关生成文本的组织、样式或布局的任何特定要求，确保输出的一致性和清晰度。

 结论：

总之，让我们比较一下 RTF（角色、任务、格式）、CTF（上下文、任务、格式）和 RASCEF（角色、操作、步骤、上下文、示例、格式）三种提示框架的优点，并讨论何时使用使用每一个。

我们还将考虑提示的类型——零样本、单样本和少样本——以及它们如何适应每个框架。

RTF（角色、任务、格式）：

 优点：

提供角色、任务和所需输出格式的清晰描述。

通过关注基本元素来简化提示设计。

对于角色和任务定义明确的简单任务很有用。

 何时使用：

非常适合具有明确定义的角色和特定目标的任务，例如生成对结构化查询的响应或完成具有明确格式要求的预定义任务。

 提示类型：

适用于所有类型的提示，包括零次、一次和几次，具体取决于任务的特殊性和所需的输出格式。

CTF（上下文、任务、格式）：

 优点：

通过将上下文信息与任务和格式要求结合起来，提供全面的方法。

帮助确保生成的响应具有相关性和情境性。

对于需要了解背景信息或情境背景的任务很有用。

 何时使用：

适用于上下文信息显着影响响应相关性的任务，或者需要额外上下文才能有效构建任务的任务。

 提示类型：

对于一次性提示和几次提示有效，提供上下文可以增强对任务的理解并提高生成响应的质量。

RASCEF（角色、操作、步骤、上下文、示例、格式）：

 优点：

提供高度结构化和详细的框架，指导从角色定义到示例说明和格式规范的提示设计过程。

提供一种全面的方法，确保任务的所有相关方面都得到考虑和解决。

对于需要分步方法、详细说明和说明性示例的复杂任务很有用。

 何时使用：

最适合涉及多个步骤、需要详细指导并从说明性示例中受益的任务。

 提示类型：

非常适合少量提示，将任务分解为步骤并提供示例可以帮助模型根据有限的输入数据生成准确的响应。

总之，RTF、CTF 和 RASCEF 提示框架之间的选择取决于任务的复杂性、所需的详细程度以及提示的类型（零样本、单样本或少样本）。每个框架都具有独特的优势，并且最适合不同的提示类型，允许提示设计的灵活性和适应性，以有效地满足特定的任务要求。
